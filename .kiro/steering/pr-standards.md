# Padrão de Pull Requests

## Título
- Formato: `[TIPO] Descrição curta` (ex.: `[FIX] Corrigir validação de CPF`)
- Tipos: FIX, FEAT, REFACTOR, DOCS, TEST, CHORE

## Descrição obrigatória
- **O que foi feito** — descrição clara da mudança
- **Por que** — contexto e motivação
- **Como testar** — passos para validar a mudança
- **Evidências** — screenshots, logs ou output de testes

## Checklist do Autor
- [ ] Código segue os padrões do time
- [ ] Testes adicionados/atualizados
- [ ] Sem secrets ou dados sensíveis no código
- [ ] Pipeline passando
