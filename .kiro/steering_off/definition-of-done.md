# Definition of Done

Uma tarefa só é considerada "done" quando:

## Código
- [ ] Código revisado por pelo menos 1 pessoa
- [ ] Sem warnings ou erros de lint
- [ ] Nomenclatura seguindo padrões do time

## Testes
- [ ] Testes unitários cobrindo happy path e edge cases
- [ ] Testes passando no pipeline

## Documentação
- [ ] PR com descrição completa
- [ ] Comentários em código complexo (quando necessário)

## Segurança
- [ ] Sem secrets hardcoded
- [ ] Inputs validados
- [ ] Dados sensíveis protegidos
